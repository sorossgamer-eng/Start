## 1. Обзор проекта

**Репозиторий:** https://github.com/sorossgamer-eng/Start

| Файл | Назначение |
|------|-----------|
| `index.html` | Лендинг услуг ремонта квартир |

| `tg-app/index.html` | Telegram Mini App — калькулятор ремонта |

## 2. Как запустить

### Вариант A — GitHub Pages (рекомендуется)
Открыть в браузере:
```
https://sorossgamer-eng.github.io/Start/index.html
https://sorossgamer-eng.github.io/Start/tg-app/index.html
```

### Вариант B — локально
1. Скачать репозиторий: `git clone https://github.com/sorossgamer-eng/Start.git`
2. Открыть любой HTML-файл двойным кликом — работает без сервера
3. Для `tg-app/index.html` — открыть напрямую в браузере (Telegram SDK gracefully деградирует)

### Требования
- Современный браузер: Chrome 90+, Safari 14+, Firefox 88+
- Интернет нужен только для `tg-app/index.html` (подключает Telegram SDK)
- Сервер не требуется

## 3. Тестирование: `tg-app/index.html`

Это основной и наиболее сложный файл. Тестировать в первую очередь.

### 3.1 Экраны и навигация

В приложении 8 экранов. Порядок прохождения основного сценария:

```
screen-onboarding → screen-welcome → screen-type → screen-params → screen-result → screen-form → screen-success → screen-portfolio
```

#### Проверить каждый экран:

| Экран | ID в коде | Что проверить |
|-------|-----------|---------------|
| Онбординг | `screen-onboarding` | Показывается при первом запуске (localStorage: `rp_onboarded`). Содержит имя пользователя из Telegram (в браузере — пусто). Кнопка «Начать →» ведёт на приветствие |
| Приветствие | `screen-welcome` | Логотип, название «РемонтПро», слоган, 4 преимущества, кнопка «Рассчитать стоимость» |
| Тип ремонта | `screen-type` | Список из 6 вариантов (см. ниже). Кнопка «Далее» заблокирована до выбора |
| Параметры | `screen-params` | Поле площади (число), 3 кнопки города, кнопка «Рассчитать» заблокирована до ввода площади |
| Результат | `screen-result` | Итоговая сумма, разбивка на 3 статьи с прогресс-барами, дисклеймер |
| Форма заявки | `screen-form` | Имя, телефон, время звонка (сегментированный контрол). Валидация |
| Успех | `screen-success` | Анимация галочки, резюме заявки, кнопка «Поделиться», кнопка «Посмотреть работы» |
| Портфолио | `screen-portfolio` | Сетка 2×2 из 4 проектов, тап открывает bottom sheet с деталями |

### 3.2 Типы ремонта и формула расчёта

```
Стоимость = Площадь × Цена_за_м² × Коэффициент_города
Исключение: Санузел — фиксированная цена 85 000 × Коэффициент_города
```

| Тип | Цена | Примечание |
|-----|------|-----------|
| Косметический | 900 ₽/м² | |
| Капитальный | 2 500 ₽/м² | Метка «Популярно» |
| Под ключ | 3 500 ₽/м² | |
| Дизайнерский | 6 000 ₽/м² | |
| Новостройка | 2 200 ₽/м² | |
| Санузел | 85 000 ₽ фиксировано | `flat: 85000` в коде |

#### Коэффициенты городов (`data-coeff` в HTML):
| Город | Коэффициент |
|-------|-----------|
| Москва | 1.3 |
| Санкт-Петербург | 1.15 |
| Другой | 1.0 |

#### Контрольные расчёты для проверки:
| Тип | Площадь | Город | Ожидаемый результат |
|-----|---------|-------|-------------------|
| Косметический | 50 м² | Другой | 45 000 ₽ |
| Капитальный | 60 м² | Москва | 195 000 ₽ |
| Под ключ | 40 м² | СПб | 161 000 ₽ |
| Санузел | любая | Москва | 110 500 ₽ |
| Санузел | любая | Другой | 85 000 ₽ |

### 3.3 Разбивка стоимости на экране результата

Результат делится на 3 статьи с прогресс-барами:
- Материалы — **40%** от суммы
- Работа — **50%** от суммы
- Накладные — **10%** от суммы

Проверить: сумма трёх статей = итоговая сумма.

### 3.4 Tab Bar (нижнее меню)

4 вкладки:
| Вкладка | Переход | Особое поведение |
|---------|---------|-----------------|
| О нас | `screen-welcome` | |
| Вид ремонта | `screen-type` | |
| Рассчитать | `screen-params` | Если тип не выбран — переходит на `screen-type` |
| Получить смету | `screen-form` | |

Проверить: активная вкладка выделена синим при смене экрана через `goTo()`.

### 3.5 LocalStorage — одноразовые элементы

Приложение запоминает 2 события в `localStorage`. Для повторного тестирования нужно удалять ключи:

| Ключ | Что блокирует | Как сбросить |
|------|--------------|-------------|
| `rp_onboarded` | Онбординг | DevTools → Application → Local Storage → удалить |
| `rp_offer_shown` | Экран-оффер | DevTools → Application → Local Storage → удалить |

**Экран-оффер:** появляется через 1200мс после загрузки. Кнопка «Получить скидку 10%» открывает `https://t.me/ремонтквартир?start=from_app` (заглушка). «Пропустить» закрывает оффер.

### 3.6 Bottom Sheet (портфолио)

При тапе на карточку проекта открывается bottom sheet с деталями. Закрывается тапом по затемнённому фону (`bs-overlay`).

### 3.7 Валидация формы заявки

Поля «Имя» и «Телефон» обязательны. При пустых полях:
- Форма не отправляется
- Поле подсвечивается анимацией `shake`

Время звонка выбирается сегментированным контролом (3 варианта). Выбор не обязателен для отправки.

### 3.8 Кнопка «Поделиться»

На экране успеха — кнопка «Поделиться с другом». Открывает:
```
https://t.me/share/url?url=https%3A%2F%2Ft.me%2Fремонтквартир&text=...
```
В браузере открывает новую вкладку. В Telegram — стандартный шаринг.

## 5. Тестирование: `crm.html`

### 5.1 Разделы

| Раздел | Навигация | Что проверить |
|--------|-----------|--------------|
| Дашборд | Клик по «Дашборд» | 4 счётчика, таблица последних клиентов |
| Клиенты | Клик по «Клиенты» | Поиск по имени, фильтр по статусу и источнику |
| Сделки | Клик по «Сделки» | Канбан-доска с 4 колонками |

### 5.2 Хранение данных

Все данные хранятся в `localStorage` под ключом `crm_clients`. После закрытия браузера данные сохраняются.

### 5.3 Добавление клиента

Кнопка «+ Добавить клиента» → модальное окно. Обязательные поля: **Имя** и **Телефон**. При пустых — алерт браузера.

### 5.4 Карточка клиента

Клик по строке таблицы → карточка клиента. Проверить:
- Смена статуса через `<select>`
- Ввод суммы сделки — сохраняется без кнопки (событие `onchange`)
- Добавление заметки — кнопка или Enter
- Удаление клиента — кнопка с подтверждением

### 5.5 Канбан-доска

4 колонки: **Новый → В работе → Оплачен → Завершён**. Карточки не drag-and-drop — статус меняется в карточке клиента через select.

## 7. Тестирование: `quiz.html`

5 вопросов с 2 вариантами ответа. Прогресс-бар сверху. После 5 вопросов — экран результата.

| Баллов | Результат |
|--------|----------|
| 4–5 | «Высокий потенциал роста» |
| 2–3 | «Есть скрытые точки роста» |
| 0–1 | «Хорошая точка для старта» |

Форма: Имя + Телефон/Email. После отправки — экран «Заявка принята» (реальная отправка не реализована, только визуал).

## 9. Известные ограничения

| Ограничение | Описание |
|-------------|----------|
| Данные только локально | `localStorage` — данные не синхронизируются между устройствами |
| Отправка форм | Реальная отправка не реализована — только визуальная анимация успеха |
| Ссылка на бота | `https://t.me/ремонтквартир?start=from_app` — заглушка, реальный бот не подключён |
| Фото в портфолио | Заглушки (эмодзи) вместо реальных фотографий |
| Отзывы | Тестовые данные, не реальные отзывы клиентов |
| Telegram SDK | В браузере деградирует корректно: haptic, BackButton и имя пользователя недоступны |

## 10. Как сообщать об ошибках

При обнаружении бага указывать:
1. **Файл** — какой HTML-файл
2. **Экран/секция** — где именно (ID экрана или название секции)
3. **Шаги воспроизведения** — что нажали, что ввели
4. **Ожидаемый результат** — что должно было произойти
5. **Фактический результат** — что произошло
6. **Браузер и размер экрана**
