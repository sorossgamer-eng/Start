# Техническое задание: Лендинг продажи услуг ремонта квартир с калькулятором

> Главный документ проекта. Основан на исследовании `research.md` и анализе лучших практик рынка.

---

## 1. О проекте

**Название проекта:** Лендинг услуг ремонта квартир
**Файл:** `remont-landing.html`
**Тип:** Одностраничный лендинг (Single Page Application)
**Технологии:** HTML + CSS + JavaScript — один файл, без внешних библиотек

---

## 2. Цель сайта

**Главная цель:** Получить заявку (имя + телефон) от потенциального клиента, который хочет сделать ремонт квартиры.

**Вторичные цели:**
- Сформировать доверие к компании через портфолио, отзывы и гарантии
- Помочь клиенту самостоятельно рассчитать приблизительную стоимость ремонта
- Выделиться среди конкурентов: фиксированная цена, штраф за просрочку, ежедневные фотоотчёты

**Метрики успеха:**
- Конверсия в заявку: целевой показатель 3–7%
- Время на сайте: более 2 минут
- Использование калькулятора: более 40% посетителей

---

## 3. Целевая аудитория

### Портрет клиента:
| Параметр | Описание |
|----------|----------|
| Возраст | 28–55 лет |
| Доход | Средний и средний+ |
| Ситуация | Купил квартиру / устарел ремонт / переезд |
| Боли | Боится переплатить, затянутых сроков, некачественной работы |
| Желания | Фиксированная цена, прозрачность, гарантия результата |
| Устройства | 65%+ — мобильные (Mobile First!) |

1. **Новостройка** — покупатели квартир без отделки, хотят въехать быстро
2. **Вторичка** — хотят обновить жильё, боятся неожиданных расходов
3. **Премиум** — ищут дизайнерское решение, качество важнее цены

### Блок 5. Калькулятор стоимости ⭐ Ключевой элемент
**Цель:** Вовлечение клиента, квалификация лида, получение контакта

**Параметры расчёта:**
- Площадь квартиры (м²) — числовое поле
- Тип ремонта — выпадающий список с ценами за м²
- Город — Москва (×1.3) / СПб (×1.15) / Другой (×1.0)

**Результат (правая колонка):**
- Общая стоимость — крупным шрифтом с градиентом
- Разбивка на 3 статьи с прогресс-барами:
  - Материалы — 40%
  - Работа — 50%
  - Накладные расходы — 10%
- Примечание: «Предварительный расчёт. Точная стоимость — после выезда замерщика»
- Кнопка: «Получить точную смету →» → модальное окно

**Формула расчёта:**
```
Стоимость = Площадь × Цена_за_м² × Коэффициент_города
```

## 6. Обязательные элементы

### Кнопки
| Тип | Стиль | Действие |
|-----|-------|----------|
| Главная CTA | Оранжевый градиент, тень, hover-эффект | Открыть модалку / прокрутить к форме |
| Вторичная | Прозрачная, белая рамка | Прокрутить к портфолио |
| В карточках услуг | Оранжевый градиент, меньше | Открыть модалку |
| В калькуляторе | Оранжевый, 100% ширина | Открыть модалку |

### Формы
- Поля с мягкими рамками, плавный focus-эффект (золотая рамка)
- Обязательная валидация: имя + телефон
- После отправки — экран-подтверждение (не перезагрузка страницы)
- На всех формах: «Без спама. Только звонок менеджера.»

### Анимации (из трендов 2024–2025)
- **Scroll-triggered:** карточки появляются снизу при скролле (opacity + translateY)
- **Счётчики:** цифры в блоке статистики считаются от 0
- **Hover на карточках:** подъём + тень (translateY -4–6px)
- **Hover на кнопках:** подъём + усиление тени
- **Модальное окно:** появление с scale(0.9 → 1) + overlay fade-in
- **Glassmorphism:** форма в Hero с backdrop-filter: blur

### Адаптивность (Mobile First)
| Брейкпоинт | Изменения |
|-----------|----------|
| < 480px | Все сетки → 1 колонка, уменьшенные отступы |
| < 600px | Статистика → 2×2, услуги → 1 колонка |
| < 760px | Преимущества → 2 колонки, отзывы → 2 колонки |
| < 860px | Hero → 1 колонка (текст сверху, форма снизу) |
| 1100px+ | Максимальная ширина контента |

## 11. План разработки Telegram Mini App — Версия 1.0

> Пошаговый план на основе `research.md`. Цель: приложение выглядит как нативное Telegram-приложение, а не как сайт, открытый в браузере.

---

### Общие требования к виду и ощущению

| Требование | Решение |
|-----------|---------|
| Нативный стиль Telegram | CSS-переменные из `tg.themeParams` — цвета берутся из темы пользователя |
| Тёмная / светлая тема | Переключается автоматически — вёрстка через `var(--tg-theme-bg-color)` |
| Крупные кнопки | Минимум 44px по высоте, ширина 100% для главных действий |
| Минимум текста | Каждый экран — одно действие, один вопрос, один шаг |
| Максимум визуала | Карточки с иконками, прогресс-бар, крупные цифры результата |
| Язык интерфейса | Русский |
| Ощущение приложения | Нет скролла там, где не нужен. Переходы между экранами — анимация слайда |

- Кнопка отправки

**Элементы:**
- Заголовок + подзаголовок
- Input «Имя» — предзаполнен из `tg.initDataUnsafe.user.first_name`
- Input «Телефон» — предзаполнен из `tg.requestContact()` (если разрешил)
- Выпадающий список / пилюли: «Сейчас» / «Утром» / «Вечером»
- `MainButton`: «Отправить заявку»
- Подпись: «Без спама. Только один звонок.»

**Действия:**
- Скорректировать имя / телефон
- Выбрать удобное время
- Нажать «Отправить заявку»

**Переход:**
- → Экран 6 (Успех)

### Экран 6 — Успех (Thank You)

- Анимированная галочка (зелёная)

**Элементы:**
- Иконка ✓ с анимацией появления (scale + fade)
- Заголовок
- Карточка-резюме с параметрами заявки
- Кнопка: «Посмотреть наши работы»
- `MainButton`: «Закрыть» → закрывает Mini App через `tg.close()`

**Действия:**

**Переходы:**

### Чего НЕ будет в версии 1.0

| Функция | Причина отсутствия |
|---------|-------------------|
| Личный кабинет / история заявок | Сложность разработки, не нужно для MVP |
| Оплата онлайн | Требует эквайринг и юридическое оформление |
| Реальный чат с менеджером | Заменяется переходом в Telegram-бот |
| Загрузка фото квартиры пользователем | Избыточно для v1 |
| Push-уведомления | Требует серверную часть |
| Слайдер «До/После» | Упрощаем портфолио до карточек |
| Мультиязычность | Только русский язык |
| Карта с выездом замерщика | Отдельная интеграция, не для v1 |
| Видеообзоры объектов | Тяжёлый контент, не для v1 |
| Личные данные пользователя в аналитике | Требует политику конфиденциальности |

## 10. Telegram Mini App — Описание экранов

> Дополнение к основному лендингу. Целевая аудитория — тёплые клиенты из Telegram-канала или бота.
> Технологии: HTML + CSS + JS + Telegram Web App SDK (`telegram-web-app.js`)

---

### Экран 1. Приветствие (Welcome Screen)

**Цель:** Быстро объяснить ценность и вовлечь пользователя

**Содержимое:**
- Логотип компании + название «РемонтПро»
- Заголовок: «Рассчитайте стоимость ремонта за 1 минуту»
- Подзаголовок: «Без звонков. Результат сразу — точная смета от замерщика бесплатно»
- Кнопка: «Рассчитать стоимость →» (главная, на всю ширину)
- Кнопка: «Посмотреть наши работы» (вторичная)
- Внизу: 3 иконки с тезисами (Договор / Гарантия 3 года / Фотоотчёты)

**Дизайн:** Тёмно-синий фон, оранжевая CTA-кнопка. Поддержка светлой и тёмной темы Telegram.

---

### Экран 2. Выбор типа ремонта

**Цель:** Квалифицировать клиента, определить бюджетный сегмент

**Содержимое:**
- Заголовок: «Какой ремонт вам нужен?»
- 5 карточек с выбором (тап = выделение):
  - Косметический — от 900 ₽/м²
  - Капитальный — от 2 500 ₽/м²
  - Под ключ — от 3 500 ₽/м²
  - Дизайнерский — от 6 000 ₽/м²
  - Новостройка — от 2 200 ₽/м²
- Прогресс-бар вверху: Шаг 1 из 3
- Кнопка «Далее →» (активна только после выбора)

**Поведение:** Выбранная карточка подсвечивается оранжевой рамкой. Одиночный выбор.

---

### Экран 3. Параметры квартиры

**Цель:** Получить данные для расчёта

**Содержимое:**
- Заголовок: «Параметры вашей квартиры»
- Прогресс-бар: Шаг 2 из 3
- Поле: Площадь квартиры (м²) — числовой ввод с плейсхолдером «например, 52»
- Выбор города:
  - Москва (коэффициент ×1.3)
  - Санкт-Петербург (×1.15)
  - Другой город (×1.0)
- Кнопки: «← Назад» и «Рассчитать →»

**Поведение:** Кнопка «Рассчитать» неактивна, пока не заполнена площадь.

---

### Экран 4. Результат расчёта

**Цель:** Показать ценность, сформировать намерение оставить заявку

**Содержимое:**
- Заголовок: «Предварительная стоимость ремонта»
- Крупная сумма с анимацией счётчика (например: **1 820 000 ₽**)
- Разбивка с прогресс-барами:
  - Материалы — 40% (сумма)
  - Работа — 50% (сумма)
  - Накладные — 10% (сумма)
- Плашка: «Это предварительный расчёт. Точная смета — после бесплатного замера»
- Кнопка: «Получить точную смету бесплатно» (оранжевая, на всю ширину)
- Ссылка: «Пересчитать» → возврат на Экран 2

**Поведение:** По нажатию главной кнопки — переход на Экран 5.

---

### Экран 5. Форма заявки

**Цель:** Получить контакт и передать лид менеджеру

**Содержимое:**
- Заголовок: «Оставьте заявку — перезвоним за 15 минут»
- Поле: Ваше имя (предзаполнено из данных Telegram)
- Поле: Телефон (предзаполнено из Telegram, если пользователь делился)
- Поле: Удобное время для звонка (выпадающий список: Сейчас / Утром / Вечером)
- Кнопка: «Отправить заявку» (оранжевая)
- Подпись: «Без спама. Только один звонок от менеджера.»

**Поведение:** После отправки — переход на Экран 6. Данные уходят в Telegram-бот менеджеру.

---

### Экран 6. Экран успеха (Thank You)

**Цель:** Подтвердить заявку, снять тревогу, предложить следующий шаг

**Содержимое:**
- Иконка: зелёная галочка (анимированная)
- Заголовок: «Заявка принята!»
- Подзаголовок: «Менеджер свяжется с вами в течение 15 минут»
- Резюме заявки: тип ремонта, площадь, город, предварительная сумма
- Кнопка: «Посмотреть наши работы» → Экран 7
- Кнопка: «Закрыть» (серая, вторичная)

---

### Экран 7. Портфолио (опциональный)

**Цель:** Удержать клиента, показать качество работ

**Содержимое:**
- Заголовок: «Наши работы»
- Сетка карточек (2 колонки):
  - Эмодзи-заглушка / фото интерьера
  - Тип ремонта, площадь, срок
- При тапе на карточку — модальное окно с деталями проекта
- Кнопка внизу: «Хочу такой же ремонт» → возврат к форме заявки

---

### Технические требования Mini App

| Параметр | Значение |
|----------|----------|
| SDK | `telegram-web-app.js` (подключается из CDN Telegram) |
| Цветовые схемы | Поддержка `colorScheme: light/dark` через CSS-переменные |
| Главная кнопка | Использовать `MainButton` из Telegram SDK для финального CTA |
| Хаптик | `HapticFeedback` при выборе карточки и успешной отправке |
| Размер | Один HTML-файл, все стили и скрипты inline |
| Хостинг | GitHub Pages (уже настроен) |
| Файл | `tma-remont.html` |
